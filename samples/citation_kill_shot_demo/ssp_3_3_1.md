# SSP §3.3.1 — Audit and Accountability (AU.L2-3.3.1)

## Implementation narrative

Hardseal creates and retains system audit logs and records to the extent
needed to enable the monitoring, analysis, investigation, and reporting of
unlawful or unauthorized system activity. Log sources are enumerated in
Policy-AC-001 v2.0 as the authoritative baseline for access-control event
capture. Supplementary guidance is provided by Runbook-IR-102 v1.0 for
incident-response log correlation workflows.

Logs are forwarded to Splunk Cloud with a 365-day retention window. The
forwarding configuration is maintained under Terraform module
`splunk_forwarders` in commit `a7f2c9d1b4`, reviewed under pull request `#482`
on 2026-02-03.
